## What this is

A personal blog published at https://ketharan.github.io via GitHub Pages, organized around three "flavours": **technical**, **travel**, and **personal**. The technical section includes a 4-part series ("Give every AI agent its own VM") whose companion code lives in a separate repo: https://github.com/Ketharan/agent-sandbox-eks.

## Build & serve

Jekyll site with **no `Gemfile`** έΑΦ it relies on the GitHub Pages build environment. To develop locally you need the `github-pages` gem (or plain `jekyll` + `jekyll-feed` + `jekyll-seo-tag`):

```bash
gem install github-pages
jekyll serve    # http://localhost:4000, live reload
jekyll build    # outputs to _site/ (gitignored)
```

There is no test suite, linter, or CI config έΑΦ pushing to `main` triggers the GitHub Pages build and deploy. (Note: modern Jekyll requires Ruby έΚξ 3.1; the GitHub Pages runner has this even though the local machine's system Ruby may be older.)

## Architecture

**Bespoke theme, no gem theme.** `_config.yml` intentionally omits a theme; all markup/styling lives here. The whole site is driven by the three categories defined once in `_config.yml` under the `sections:` key (`slug` / `label` / `blurb`). It's keyed `sections` rather than `categories` because `site.categories` is Jekyll-reserved. Layouts iterate `site.sections` to build nav, the home page, and footer έΑΦ **add a flavour by adding an entry there** (plus a `<slug>.md` category page and a `.cat-<slug>` color block in the CSS).

**Categories are driven by each post's `category:` front-matter**, and `permalink: /:categories/:slug/` in `_config.yml` gives posts category-prefixed URLs (e.g. `/travel/kyoto/`). CSS uses a per-category accent via the `.cat-<slug>` class + a `--accent`/`--accentbg` CSS-variable pair; markup applies `cat-{{ category }}` on cards, heroes, and posts.

Layouts (`_layouts/`):
- `default.html` έΑΦ shell (header nav, footer, fonts, reading-progress bar); both are generated by looping `site.sections`.
- `home.html` (`index.md`) έΑΦ hero + one section per flavour, each showing its 3 latest posts with a "view all" link.
- `category.html` έΑΦ the `/technical/`, `/travel/`, `/personal/` listing pages (`technical.md` etc., each sets `layout: category` + `category:` + `permalink`). Renders any **series** in that category as a grouped block first (assumes one series per category), then standalone posts.
- `post.html` έΑΦ article layout. Series behavior (the series pill and prev/next nav) is now **conditional on `page.series`** and fully generic: prev/next are resolved from sibling posts sharing the same `series` value, sorted by `part` έΑΦ no hardcoded part count or URLs. Non-series posts just get a category pill and a "More <category>" back-link.
- `_includes/post-card.html` έΑΦ the shared post card used by home + category pages.

**Post front-matter contract:**
- Every post needs `category:` (one of the `sections` slugs) έΑΦ this sets its URL and accent color.
- **Series posts** additionally carry `series:` (display name, identical across the set), `part:` (integer), an explicit `permalink:` (the series predates category URLs έΑΦ the 4 tech posts keep `/part-N/`), and optional `repo:` (shown as a "Companion repo" link in the post footer).
- Titles use the form `"Part N έΑΦ έΑο"` / `"έΑο έΑΦ έΑο"`; layouts `split: 'έΑΦ '` to strip any prefix before the em dash.

**Content conventions** (see the technical series): posts are written to be followed by a human *or* an agent. Inline markers έΑΦ `<PLACEHOLDER>`, `έεΖ Verify` gates, `ΏθΡδ What bit me` boxes, `ΏθπΣ Human step` / `ΏθνΨ Scriptable` tags έΑΦ are color-coded from their leading emoji by `assets/js/main.js` (`.prose blockquote` έΗΤ `.cl-*` classes), so keep the emoji/format consistent. The same JS adds copy buttons to code blocks and drives the progress bar. OpenChoreo Helm charts in the series are pinned to `0.0.0-c6ce5916`; keep pins consistent across parts.

**Example posts:** `_posts/2026-07-15-example-travel-post.md` and `2026-07-18-example-personal-post.md` are placeholder/template posts (tagged `example`) so the Travel/Personal sections aren't empty έΑΦ edit or delete them when real content lands.

**Plugins:** `jekyll-feed` (`/feed.xml`) and `jekyll-seo-tag` (the `{% seo %}` tag in `default.html`).
